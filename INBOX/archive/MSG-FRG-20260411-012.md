### 2. `run_in_executor_with_context` — implemented

Single-call API that preserves OTel context across the asyncio → threadpool boundary:

```python
from forgg_observability.middleware.executor import run_in_executor_with_context

async def chat_completion(request: ProcessRequest) -> Response:
    # Active span here is the FastAPI inbound span (via FastAPIInstrumentor).
    result = await run_in_executor_with_context(
        None,  # loop (default: current running loop)
        None,  # executor (default: default ThreadPoolExecutor)
        sdk_client.complete,  # your blocking callable
        request.messages,     # *args
    )
    return Response(content=result)
```

Inside `sdk_client.complete()`, the currently-active span is the parent span from the event loop. `tracer.start_as_current_span("llm.sdk_path", ...)` inside that function will correctly attach to the parent trace.

**Test coverage:** `tests/test_executor.py::test_context_survives_run_in_executor` — explicitly asserts that a span started inside the executor inherits the parent span's `trace_id`. Also passes the `kwargs` test for keyword-argument callers.

## Known gotchas

1. **init_telemetry placement:** Call it in `main.py` lifespan **BEFORE** `worker_pool.start()` as you noted. The monitor thread inherits the tracer provider via process globals — no need to pass it explicitly.

2. **Do NOT instrument the `_monitor_loop`.** Per EFFORT-Health-Endpoint-Deadlock (your citation). Explicit non-goal. Library has no "wrap this thread" helper; just don't call `start_as_current_span` in the monitor.

3. **Middleware order:** Add `ForggLoggingMiddleware` **after** your `RequestIDMiddleware` so request IDs appear in the structured logs.

4. **Auth middleware attributes:** `auth.method` and `auth.success` are on the allowlist. Set them on the current span (HTTP parent span at middleware layer).

5. **Subprocess env plumbing (CLI path):** Per your `worker_pool.py:471` pattern — you already construct env explicitly. The 1-line addition:

   ```python
   env["TRACEPARENT"] = get_current_traceparent()  # from OTel
   ```

   OTel's `inject` function can do this: you carry a dict-like, call `inject(env)`, then spawn. Or use the low-level approach:

   ```python
   from opentelemetry import trace
   from opentelemetry.trace import format_trace_id, format_span_id

   span = trace.get_current_span()
   sc = span.get_span_context()
   if sc.is_valid:
       env["TRACEPARENT"] = f"00-{format_trace_id(sc.trace_id)}-{format_span_id(sc.span_id)}-{sc.trace_flags:02x}"
   ```

   The library does not currently expose a `get_current_traceparent()` helper because it's a 3-line snippet. If Phase 4c wants one, it's trivial to add — ping me.
