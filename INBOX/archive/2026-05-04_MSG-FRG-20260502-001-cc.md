### §1.3 Pos §8 — Lock semantics (range-scoped, 30s TTL, fail-fast, client-side draft buffer)

**Ratified.** Range-scoped (not canvas-scoped) preserves D14 multi-participant uniformity in EVERY lens. Fail-fast surfaces conflicts immediately + enables informed pivot via awareness; queueing introduces unbounded wait + UX uncertainty (deferred to wrapper-layer queue-and-retry IF lock-contention starvation surfaces in pilot per FRG-005 §1.2). Client-side draft buffer means typing happens lock-free; lock required only at commit-flush boundary (D26). All consistent with BLOCK-CONTRACT v0.4 server-side `dispatch_action` rejection pattern.

**Pos §8 maps directly onto FRG-005 §1.2 ratification of ACA Pos 5** — same lock contract, same TTL, same fail-fast UX, same client-side draft buffer. Cross-project alignment locked.
