## Behavior on the wire (what ACA will see)

| Surface | Value |
|---------|-------|
| Response header `X-Forgg-CCA-Request-Id` | per-request UUID, present on every CCA `/v1/*` response (including `/v1/messages` equivalent endpoints, `/v1/chat/completions`, `/v1/process`, `/v1/task`, `/health`) |
| Response header `X-Request-Id` | same UUID — single mint, two header aliases |
| Span attribute `forgg.cca_request_id` | set on every span emitted during the request — FastAPI HTTP parent (from FastAPIInstrumentor) plus `gen_ai.*` SDK-path spans plus WorkerPool CLI-path spans |
| Inbound `X-Request-Id` | honored if a client sent one (no double-mint) |

## How it's wired (FYI)

Implementation pattern: contextvar (`cca_request_id_ctx`) populated by `RequestIDMiddleware` + custom `CCARequestIdSpanProcessor.on_start()` registered with the global tracer provider. The processor copies the contextvar onto every recording span, so the attribute lands on **all** spans in the trace regardless of which instrumentor created them.

Code is at `src/middleware.py` (RequestIDMiddleware + CCARequestIdSpanProcessor) and `main.py` lifespan (registration). 8 new tests in `tests/test_middleware.py` cover the contract end-to-end including a real `TracerProvider` + `InMemorySpanExporter` integration test. Full suite green: 333 passed, 16 skipped.
